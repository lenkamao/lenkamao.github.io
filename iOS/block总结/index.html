什么是 Block?

Blcok是将函数及其执行上下文封装起来的对象.

  • blcok 类似于 JAVA 中的钩子函数,他不仅包含毁掉期间的代码,还有执行期间的数据
  • blcok 支持多线程
  • blcok 除了执行代码外,还有堆栈的变量

  • block 的形式有以下几种:

  • block 自定义方式

Block 截获变量

  • block 允许对外部变量进行读取操作,不允许写操作

  • block 可以对声明为__block类型的变量进行写操作

  • block 截获局部变量(基本数据类型和对象类型)

    • 对于基本数据类型的局部变量获取其值
    • 对于对象类型的局部变量连同所有权修饰符一起获取
  • block 截获静态局部变量
    • 对于静态局部变量以指针的形式获取
  • block 截获全局变量
    • 想多了,并不截获
  • block 截获静态全局变量
    • 想多了,并不截获

Block 的修饰符

  • block修饰符,一般情况下,对被截获变量进行赋值即写操作时候需要添加 block修饰符,注意:赋值和使用是不同的.
  • 局部变量需要修饰符
  • 非局部变量不需要修饰符
  • __block 修饰的变量变成了对象

__forwarding指针指向自身获取变量(栈区)

Blcok 内存管理

  • 全局 Blcok(NSConcreteGlobalBlock),存在于全局内存中, 相当于单例.
    • block不访问外接变量的时候, block 既不在栈中也不在堆去,在代码段中
  • 栈 Blcok(NSConcreteStatckBlock),存在于栈内存中, 超出其作用域则马上被销毁.
    • MRC 环境下：访问外界变量的 Block 默认存储栈中。
    • 注意: 栈中的 block 随时可能销毁
  • 堆 Blcok(NSConcreteMallocBlock),存在于堆内存中, 是一个带引用计数的对象, 需要自行管理其内存
    • ARC 环境下：访问外界变量的 Block 默认存储在堆中（实际是放在栈区，然后ARC情况下自动又拷贝到堆区），自动释放。
    • 通过forwarding, 无论是在block中还是 block外访问block变量, 也不管该变量在栈上或堆上, 都能顺利地访问同一个__block变量。
